## 📐 物理・数学の復習

### 1. 等速運動（速度一定）

物体の位置 \(\vec{p}\) と速度 \(\vec{v}\) の関係：

\[
\vec{p}_{new} = \vec{p}_{old} + \vec{v}
\]

成分で書くと：
\[
\begin{aligned}
x_{new} &= x_{old} + v_x \\
y_{new} &= y_{old} + v_y
\end{aligned}
\]

**例題：**
- 現在位置: (100, 100)
- 速度: (5, 10)
- 1フレーム後の位置: (100 + 5, 100 + 10) = (105, 110)

### 2. 加速度運動（速度が変化）

加速度 \(\vec{a}\) による速度の変化：

\[
\vec{v}_{new} = \vec{v}_{old} + \vec{a}
\]

重力の場合、加速度は常に下向き（ワールド座標では負のy方向）：
\[
\vec{g} = (0,\ -g)
\]

**例題（重力 g = 0.3）：**
- 現在の速度: \(v_y = +10\) (上向き)
- 1フレーム後: \(v_y = 10 + (-0.3) = 9.7\)
- 2フレーム後: \(v_y = 9.7 + (-0.3) = 9.4\)
- ...
- Nフレーム後: \(v_y = 0\) (最高点)
- N+1フレーム後: \(v_y = -0.3\) (下降開始)

### 3. 放物線運動の仕組み

**手順：**
1. 速度に重力を加える → 速度が変化
2. 位置に速度を加える → 位置が移動

```
毎フレーム:
  ステップ1: v = v + g  (速度更新)
  ステップ2: p = p + v  (位置更新)
```

**なぜ放物線になるのか？**
- x方向: 速度が一定 → 等速直線運動
- y方向: 速度が変化 → 加速度運動
- 結果: x-y平面上で放物線を描く

**ステップ1: 速度の更新**
\[
\vec{v}_{new} = \vec{v}_{old} + \vec{g}

**ステップ2: 位置の更新**

\vec{p}_{new} = \vec{p}_{old} + \vec{v}_{new}

```cpp
void Bullet::Update()
{
    if (!isActive_) return;  // 無効な弾は更新しない

    // ステップ1: 重力を速度に加える
    
    // 重力ベクトルを作る（ワールド座標では下向き = 負のy）
    Vector2D gravityVec = { 【⑥】, 【⑦】 };
    
    // 速度 = 速度 + 重力
    vel_ = Math2D::【⑧】(【⑨】, gravityVec);

    // ステップ2: 速度を位置に加える
    
    // 位置 = 位置 + 速度
    pos_ = Math2D::【⑩】(【⑪】, 【⑫】);

    // 画面外判定（提供済み）
    if (pos_.x < 0 || pos_.x > WIN_WIDTH || 
        pos_.y < 0 || pos_.y > WIN_HEIGHT)
    {
        isActive_ = false;
    }
}
```

**ヒント：**
- ⑥: 重力のx成分（横方向の重力はない）
- ⑦: 重力のy成分（下向き = 負、重力加速度は `gravity_` 変数）
- ⑧: ベクトルを足す関数（Math2D の関数）
- ⑨: 現在の速度を表すメンバ変数
- ⑩: ベクトルを足す関数（Math2D の関数）
- ⑪: 現在の位置を表すメンバ変数
- ⑫: 更新された速度を表すメンバ変数

**重要な注意：**
- ワールド座標では y軸が**上向き**
- 重力は**下向き**なので、y成分は**負の値**
- 重力の大きさは `gravity_` で、符号を考慮すると `-gravity_`

| フレーム | v_x | v_y | x | y | 説明 |
|---------|-----|-----|---|---|------|

| 2 | 10.0 | 9.4 | 20 | 19.7 | 上昇中 |

**この表から分かること：**
- v_x は常に一定（10.0）
- v_y は毎フレーム 0.3 ずつ減少
- x は等速で増加
- y は最初速く増加し、徐々に遅くなり、最高点で止まり、その後下降

## ?? デバッグのヒント

もし弾の動きがおかしい場合は、以下を確認してください：

1. **弾がまっすぐ飛ぶ（放物線にならない）**
   - → 重力ベクトルの y 成分が 0 になっていないか確認
   - → `vel_` の更新を忘れていないか確認

2. **弾が真上に飛ぶ**
   - → 重力の符号が逆（正の値）になっていないか確認
   - → ワールド座標では下向き = 負の値

3. **弾が瞬間移動する**
   - → `Math2D::Add()` の引数の順序を確認
   - → 位置と速度を正しく渡しているか確認
